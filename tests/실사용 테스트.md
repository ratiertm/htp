## L1 → L2 분기점 검증 시나리오 (sidequest 인수기준)

### S1. Batch ingest — Obsidian Daily Notes 일괄
**Use case**: Obsidian 의 매일 메모 30일치를 한 번에 ingest.

```bash
# 현재 (불가능):
for f in ~/Documents/Obsidian/Daily/*.md; do
    python -m htp.knowledge ingest --source obs "$(cat $f)"
done  # 30번 프로세스 spawn → encoder.fit() 30번 + 매번 cache reload

# 목표 (L2):
python -m htp.knowledge ingest --dir ~/Documents/Obsidian/Daily \
    --source obs --pattern "*.md"
```

**Pass 기준**:
- 한 프로세스로 N 파일 처리 (encoder.fit() 1회만)
- 각 파일이 별도 entry 로 저장
- 진행률 표시 (`[12/30] daily-2026-04-15.md ✓`)
- 실패 파일 skip + 마지막에 에러 요약

### S2. Stdin pipe — shell 통합
**Use case**: 회의 메모를 받아쓰기 후 즉시 ingest.

```bash
pbpaste | python -m htp.knowledge ingest --source meeting --tag urgent
# 또는
python ~/scripts/web-extract.py "https://..." | python -m htp.knowledge ingest --source web
```

**Pass 기준**:
- `--text` 인자 없으면 stdin 사용
- pipe 빈 입력 시 안전한 에러
- 출력은 한 줄 (`✓ saved (id=42, source=meeting)`)

### S3. Source / time filter — 특정 도메인 검색
**Use case**: "지난 한 달간 brain 도메인에서 '패턴 인출' 관련 메모".

```bash
python -m htp.knowledge query "패턴 인출" --source brain --since 30d
python -m htp.knowledge discover --source-pair brain,ai --since 2026-04
```

**Pass 기준**:
- `--since` 가 ISO 날짜 / `Nd` (N days ago) 둘 다 지원
- `--source` 미일치 entry 는 검색 대상에서 제외
- filter 적용 후 빈 결과 시 명확한 메시지

### S4. Edit / delete / tag — entry 관리
**Use case**: 잘못 저장된 entry 삭제, 사후 태그 추가.

```bash
python -m htp.knowledge list --source brain --limit 10           # id 확인
python -m htp.knowledge delete --id 7                            # 삭제
python -m htp.knowledge tag --add "memory,distributed" --id 4    # 태그 추가
python -m htp.knowledge edit --id 4 --text "수정된 내용..."       # 본문 수정
```

**Pass 기준**:
- JSONL append-only 깨지 않게 tombstone 패턴 또는 compaction
- delete 후 query/discover 에서 미반환
- edit 시 timestamp 와 vec 재계산 (id 유지)
- tag 검색 가능 (`--tag memory`)

### S5. Export — 다른 도구로 내보내기
**Use case**: 한 달치 knowledge 를 markdown 으로 정리해서 보고서로.

```bash
python -m htp.knowledge export --format markdown \
    --since 2026-04 --group-by source > monthly.md
python -m htp.knowledge export --format json --source brain > brain.json
python -m htp.knowledge export --format obsidian \
    --dir ~/Documents/Obsidian/HTP-Export/
```

**Pass 기준**:
- markdown: source 별 섹션, timestamp 정렬
- json: 원본 vec 보존 + 메타데이터 포함
- obsidian: 파일 단위 split + frontmatter (source/tags/created)

## L2 → L3 분기점 검증 시나리오 (sub-5 EmbeddingBridge 자동 도달)

### S6. 한국어 의미 매칭
**Use case**: "기억은 분산이다" 와 "기억이 분산되어 있다" 가 같은 의미로 매칭.

```bash
python -m htp.knowledge ingest --source brain "기억은 분산 표상이다"
python -m htp.knowledge query "기억이 분산되어 있다"
# 기대: similarity ≥ 0.7
```

**현재 (L1/L2)**: TF-IDF 형태소 미분리 → `기억은`/`기억이` 다른 토큰 → similarity ≈ 0.0
**L3 (sub-5)**: 사전학습 한국어 임베딩 (`bge-small-ko-v1.5`) → 의미 매칭 ✅

### S7. Cross-language hub
**Use case**: "attention mechanism" 영문 메모와 "어텐션 메커니즘" 한국어 메모가 연결.

```bash
python -m htp.knowledge ingest --source ai-en "attention mechanism distributes weights"
python -m htp.knowledge ingest --source ai-ko "어텐션 메커니즘은 가중치를 분산한다"
python -m htp.knowledge discover --threshold 0.5
# 기대: ai-en ↔ ai-ko 연결 발견
```

**현재**: 영문 술어가 한국어 문장에 섞일 때만 부분 hub (sub-1 실험에서 검증)
**L3**: bilingual 임베딩으로 진정한 cross-language 연결

## L3 → L4 분기점 검증 시나리오 (별도 cycle `htp-knowledge-integration`)

### S8. Obsidian 양방향 동기
**Use case**: Obsidian vault 의 메모와 HTP knowledge 가 양방향 sync.

```bash
htp-knowledge sync obsidian --vault ~/Documents/Obsidian --mode bidirectional
# 신규 메모 → knowledge 자동 ingest
# knowledge tag 변경 → obsidian frontmatter 반영
```

### S9. LLM query 통합
**Use case**: LLM 답변에 knowledge_log 컨텍스트 자동 주입.

```bash
htp-knowledge ask "Hopfield network 와 인간 기억의 차이는?"
# → query "Hopfield" + query "인간 기억" → top-3 entries 추출
# → LLM prompt 에 컨텍스트로 주입 → 답변
```

### S10. Daily 자동화
**Use case**: 매일 자동 ingest + summary.

```bash
# crontab
0 23 * * *  htp-knowledge daily-ingest --source obs && \
            htp-knowledge daily-summary --send-to slack
```
